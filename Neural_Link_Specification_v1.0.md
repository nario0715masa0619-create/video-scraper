# 論理接続仕様書（Neural Link Specification v1.0）

## 1. 接続トリガーの定義 (Connection Triggers)

Neural Networkingにおいて、単純なキーワードの完全一致（例：「リスト取り」「LINE誘導」）だけでなく、以下の「文脈的類似性（Contextual Similarity）」をフックとしてエッジ（繋がり）を形成しました。

- **目的と解決策の合致 (Goal-Solution Matching)**: 一方の要素が「課題・目標（例：コンテンツ作成が続かない）」を提示し、他方がその「実践的解決策（例：AIと文字起こしを使った量産体制）」を提示している場合。
- **プロセスチェーンの順序性 (Process Sequentiality)**: DRM（集客・教育・販売）のような一連のビジネス構造において、「Aが完了しなければBに移行できない」という時間的・工程的な連続性。
- **抽象から具象へのブレイクダウン (Abstraction to Concretion)**: 「属人性の排除」という抽象的なマインドセット（上位概念）と、「クラウドソーシングでの文字起こし発注」という具体的なSOP（下位概念）の包含関係。

## 2. relation_type（依存/手段/制約）の判定ロジック

ノード間の関係性を3つのカテゴリに分類する際、AI推論（Gemini 3 Pro）内部では以下の明確な閾値と基準をもって判定を下しました。

- **依存 (Dependency) - [不可分・前提条件のつながり]**:
  - **判定基準**: 要素Bを実行・理解するために、要素Aが「不可欠な前提知識」または「物理的な前工程」である場合。
  - **閾値**: Aが欠落した場合、Bが成立しない、または致命的な欠陥を生むか？（Yesなら「依存」）。
  - **例**: 「LINE公式アカウントの開設（A）」は「ステップメールによる教育（B）」に対する【依存】。

- **手段 (Means) - [目的と実行方法のつながり]**:
  - **判定基準**: 要素A（目的・戦略）を達成するための、複数の選択肢のうちの「具体的な実行アプローチ・ツール」が要素Bである場合。
  - **閾値**: BはAを達成するための「唯一絶対の方法」ではなく、「取り得る有効なアクションの一つ」であるか？（Yesなら「手段」）。
  - **例**: 「外注化・組織化（A）」という目的に対する、「ココナラでのサムネイル発注（B）」は【手段】。

- **制約 (Constraint) - [ルール・警告・フェーズ制限のつながり]**:
  - **判定基準**: 要素Bが要素Aを実行する上での「守るべき禁止事項」「リスク」「条件」、または「矛盾を含むため注意すべき事項」である場合。
  - **閾値**: Bを加味せずにAを実行すると、アカウント凍結や資金ショート等の不利益が生じるか？ またはAの実行を躊躇させる競合情報か？（Yesなら「制約」）。
  - **例**: 「Brainでの販売（A）」に対する「限定価格表記の規約違反（B）」は【制約】。

## 3. 一貫性バリデーションの推論フロー（事業投資ロジックの解決）

「月10万稼ぐ（初期目標: 動画03）」という文脈に対し、「外注費に数万〜10万円かける」といった一見相反する高度なノウハウが提示された場合、システムは以下の思考ステップで矛盾を解消し、論理破綻を防ぎました。

1. **ベースコンテキストの定義（前提の認識）**:
   - ユーザーの初期状態を「資金ゼロ・知識ゼロから初めの0→1（最初の1万円〜10万円）を目指すフェーズ」と最優先コンテキストに設定。
2. **リソース要求の評価（矛盾の検知）**:
   - 各SOPが要求するリソース（資金・時間・スキル）を評価。「高額な外注費」は初期の資金力レベル（前提）と衝突することを検知。
3. **フェーズ分離による解決（推論の実行）**:
   - 「高額外注は間違っている」と情報を破棄するのではなく、**「ビジネスの発展フェーズが異なる」**と推論。
4. **論理エッジの構築**:
   - 初期目標ノードから高額外注ノードへの繋がりを「制約（Constraint）」として設定。
   - `reason_logic`（推論根拠）に、「**この手法はスケールアップ時（月10万達成後）のフェーズに適用すべきであり、0→1の初期段階においては資金的リスクとなるため、動画07の低コスト手法（1,000円外注）を優先実行すべきである**」という解決ロジックを言語化してメタデータとして記録。

## 4. エビデンス優先順位の重み付け（マスターデータ選定基準）

複数の動画で重複するテーマ（例：サムネイル作成手法）が語られた際、情報の優劣（`integrated_importance_score` の算出および `warning_flag` での優先度提示）を決定する重み付けは以下のアルゴリズムに従いました。

1. **具体性のスコア（最重視）**:
   - 抽象的な概念（「安く発注する」）よりも、明確な数値・プラットフォーム名・プロンプト・相場価格（「ココナラで1,000円で発注する」）を含む要素を圧倒的に優先。
2. **再現性・SOP化のスコア**:
   - 個人の才能に依存する属人的な「FACT（事実）」よりも、誰でもそのまま実行できる手順書化された「SOP（手順）」の要素を優先。
3. **情報の更新性（最新優先法則）**:
   - シリーズ動画において、後続の動画（例：動画08）で語られる内容は、より文脈が深掘りされ、かつ最新のアルゴリズムや市場環境を反映していると仮定し、微細な矛盾がある場合は後続情報を優先基準とする（ただし、前述の初期フェーズ制約に該当する場合は例外）。
4. **致命的リスク回避のスコア**:
   - 「アカウントBAN」や「赤字リスク」など、ビジネスの生存に関わる警告情報（制約）を含む要素は、無条件で優先度を跳ね上げ、必ず他の関連ノウハウに「制約エッジ」として紐付くよう重み付けを行った。
