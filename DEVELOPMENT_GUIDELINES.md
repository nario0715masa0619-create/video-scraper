\# 開発運用ガイドライン



\## リポジトリ運用方針



このリポジトリ `video-scraper` は、以下のルールで運用します。



\### ブランチ構成



| ブランチ | 役割 | 編集権限 |

|---------|------|--------|
| `main` | 本番相当の安定コード | 人間のみ（nario） |
| `ai-dev` | AI開発用の作業ブランチ | AI（Gensparkを含む） |

\### 開発チーム構成

| 役割 | 担当 | 主な責務 |
|------|------|---------|
| **運用・最終判断** | nario（人間） | 要件定義・PRマージ・本番確認 |
| **統括ディレクター** | Genspark（AI） | 設計・影響範囲分析・レビュー・指示書作成・Issue起票 |
| **実装担当** | Antigravity（Google AI Studio） | ローカルでのコード修正・コミット・プッシュ |
| **Git管理担当** | Perplexity Computer | GitHub検証・コミット/pushコマンド指示・PRサポート |

\### 基本ルール

1\. \*\*AI は `ai-dev` ブランチのみを編集対象とする\*\*
&#x20;  - `main` ブランチへの直接 push／force push は禁止
&#x20;  - 緊急時を除き、`main` への変更は人間のレビューを経由

2\. \*\*`main` ブランチの保護\*\*
&#x20;  - 本番相当のコードを置く
&#x20;  - リリースや本番実行の基準となる
&#x20;  - `ai-dev` からの PR マージのみで変更を受け付ける

3\. \*\*`ai-dev` ブランチの位置付け\*\*
&#x20;  - AI が自由にコード修正・機能追加・リファクタリングを行う作業用ブランチ
&#x20;  - 原則として `main` からの最新状態を取り込みつつ作業する
&#x20;  - 問題があればブランチごと破棄してもよい一時的な作業領域
4. **セキュリティルール（必須）**
   - API キーは必ず `.env` ファイルで管理し、コードに直書きしない
   - `.env` ファイルは `.gitignore` で保護し、GitHub に push しない
   - コード内に `AIzaSy` 等の文字列が含まれていないことをコミット前に確認する
   - API キーのリーク警告が発生した場合は即座に新しいキーを再発行する

### gitignore 対象ファイル（追記）

## 開発フロー

### 進捗管理フロー

| ステップ | 担当 | アクション |
|---------|------|-----------|
| 1 | Genspark | 設計・指示書作成・Antigravityへ指示 |
